# demo-data

Seed source files for manual testing and demoing the ingestion pipeline.

Since Milestone 3, these files are live demo content: upload any of them
through the Documents screen (or `POST /api/v1/documents`) to see the
full pipeline run end to end -- extraction, chunking, embedding,
classification (Milestone 6), concept linking (Milestone 7), and
retrieval with provenance (Milestone 8).

## Files

- `Expense_Policy.pdf`, `Employee_Handbook_Excerpt.pdf`,
  `Vendor_Contract_Summary.pdf` -- three generic corporate-policy PDFs,
  generated by `generate_demo_pdfs.py` (requires PyMuPDF).
- `data_retention_policy.docx` / `.pptx` / `.md` / `.py` / `.png` --
  five fixtures covering the remaining supported formats, generated by
  `generate_demo_fixtures.py` (requires python-docx, python-pptx,
  Pillow). Deliberately share one filename stem so, once ingested, they
  dedupe into a single concept via `LocalConceptLinker`'s
  filename-fallback mechanism -- see that script's module docstring for
  the full mechanism and `app/services/concept_linking.py`. This is the
  Milestone 12 (Section 4.3) cross-format concept-linking demonstration:
  one concept, "Data Retention Policy", with real evidence from five
  different source types.

Both generator scripts are one-off content generators, not run by CI or
any test -- their *output* (the files themselves) is what's checked in
and what `seed.py` uploads.

## Seeding a workspace

With the API running (`docker compose up`), populate a fresh workspace
through the real endpoints (registration, upload, polling -- no database
shortcuts):

```
python3 demo-data/seed.py
```

See `seed.py --help` for the demo account's email/password/base-URL
overrides. Re-running is safe -- already-uploaded files are reported as
skipped (`409 DUPLICATE_DOCUMENT`) rather than re-uploaded.

The eighth demo entry, a YouTube video transcript (the seventh supported
source type overall), is **not** included in `seed.py` -- see
`YOUTUBE_REFERENCE.md` for why and how to ingest one manually.
